# Repository Agent Rules

本文档告诉进入本仓库工作的 AI agent 需要遵守的规则。

目前主要针对这个问题：在没有额外说明时，应如何放置和命名文档。

## 默认环境规则

1. 本仓库默认使用的 conda 环境名为 `qis`。
2. 只要涉及 Python 脚本运行、依赖安装、notebook 内核或环境说明，默认都以 `qis` 作为标准环境名。
3. 若用户明确指定其他环境名，以用户要求为准。

## 文档放置规则

1. 只要是 AI 新写的文档，除非用户明确要求放到别处，否则默认放在 `docs/notes/`。
2. AI 不应把自己写的研究说明、实验方案、阶段总结默认放到 `docs/reference/`。
3. `docs/reference/` 只用于外部资料整理、规则摘录、接口说明或原始参考文档。

## 文档命名规则

1. AI 生成文档默认命名为：
   - `日期_AI名_文档名.md`
   - 例如：`0402_GPT_GCMI参数扫描实现方案.md`
2. 如果文档明确是用户本人撰写或由用户维护，则命名为：
   - `日期_文档名.md`
3. 这里的“日期”默认使用 `MMDD` 四位格式。
4. “AI名”使用当前 agent 最常见、最短的标识即可，例如：
   - `GPT`
   - `Codex`

## 执行优先级

1. 用户显式指定路径或文件名时，以用户要求为准。
2. 用户没有指定时，严格遵守本文件中的默认规则。

## 路径输出规则

1. 所有面向用户的路径输出、文件引用、目录说明，默认使用仓库根目录下的相对路径。
2. 除非用户明确要求，否则不要输出绝对路径。

## 适用范围

1. 本规则适用于当前仓库内所有新建文档。
2. 本规则重点约束 `.md` 文档；除非用户明确要求，不要把这类说明性内容写到仓库根目录。
